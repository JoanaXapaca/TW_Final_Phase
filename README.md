#READ.ME Entrega 2

No que diz respeito à componente de investigação, foi implementado um slider interativo que permite uma navegação dinâmica pelos dados recolhidos. Esta abordagem permite ao utilizador percorrer as informações de forma intuitiva. A secção de parcerias foi reforçada com a inclusão de um carrossel de logótipos, otimizando o espaço vertical da página e garantindo que todos os colaboradores externos tenham o devido destaque visual.

Para garantir a coerência estética em toda a experiência de navegação, a paleta de cores do site foi ajustada, adotando o azul como cor predominante. Esta alteração assegura a consistência visual entre as diferentes secções e reforça a identidade do projeto. No âmbito da usabilidade, foram integradas funcionalidades de acessibilidade, como o botão "Scroll to Top", que permite um retorno rápido ao topo da página.

A nível de animações, foi feito o uso de bibliotecas externas:

- Biblioteca Grasp: Utilizada para a implementação de efeitos de fade-in e fade-out na transição entre secções.

- Biblioteca Chart.js: Aplicada na renderização de gráficos, garantindo que os dados  sejam apresentados de forma clara e visualmente apelativa.

#READ.ME Entrega 3

A landing page do CACA foi expandida com a implementação de um sistema de gestão de eventos (CRUD) utilizando IndexedDB para persistência local, permitindo manipular eventos que ficam guardados entre sessões.
Foi também adicionado um formulário de subscrição de newsletter com validação e armazenamento em IndexedDB. Foram integradas três APIs externas:
- API de previsão meteorológica (wttr.in) para consultar condições climáticas de eventos;
- API de mapas (OpenStreetMap com Leaflet.js) para exibir localizações num mapa interativo;
- RSS feeds públicos da área da saúde para exibir notícias relevantes.
Todas as novas funcionalidades mantêm a identidade visual e o design responsivo das entregas anteriores.
